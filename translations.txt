TRANSLATIONS — PHASE 8
======================

How it works
------------

The plugin strings are written in Dutch (the source language in the code).
WordPress treats those strings as the source and looks up a translation in a
.mo file based on the active site locale.

- Site locale is nl_NL  →  no .mo file needed  →  Dutch strings render from code
- Site locale is en_US  →  WordPress loads ftb-donation-form-en_US.mo  →  English shown

With WPML or TranslatePress, the locale switches automatically per page or per
visitor. The plugin does not need any special handling — load_plugin_textdomain()
runs on plugins_loaded before any output, so the correct .mo file is already
loaded when the shortcode renders.


Files per language
------------------

For every language you want to support, two files go in /languages:

  ftb-donation-form-en_US.po   ← human-editable source (translated in Poedit)
  ftb-donation-form-en_US.mo   ← compiled binary that WordPress reads

The .po is created from the .pot template. Poedit compiles the .mo automatically
when you save.


What is already done
--------------------

- All strings in PHP are wrapped in __() / esc_html__() / esc_html_e() with
  the text domain 'ftb-donation-form'
- JavaScript error messages are passed from PHP via wp_localize_script, so they
  are covered by the PHP i18n functions — no separate JS translation file needed
- Domain Path: /languages is declared in the plugin header
- load_plugin_textdomain() is implemented in class-ftb-donation-form-i18n.php
- The .pot template has been generated with 100 translatable strings


How to add a new language (e.g. English)
-----------------------------------------

Step 1 — Open Poedit (free at poedit.net)

Step 2 — File → New from POT/PO file
         Select: languages/ftb-donation-form.pot

Step 3 — Choose the target language (e.g. English / en_US)

Step 4 — Translate all 100 strings

Step 5 — Save
         Poedit writes:
           languages/ftb-donation-form-en_US.po
           languages/ftb-donation-form-en_US.mo

Step 6 — Test
         In WordPress go to Settings → General → Site Language
         Switch to English and reload the form page


How to update the .pot after code changes
------------------------------------------

Whenever new strings are added to the plugin, regenerate the .pot with WP-CLI.
WP-CLI is not installed globally on this machine, but Local's PHP binary can
run it directly.

Run this command from the plugin root:

  "C:\Users\rosit\AppData\Roaming\Local\lightning-services\php-8.2.29+0\bin\win64\php.exe" ^
  -c "C:\Users\rosit\AppData\Roaming\Local\lightning-services\php-8.2.29+0\bin\win64\php.ini-production" ^
  -d "extension_dir=C:\Users\rosit\AppData\Roaming\Local\lightning-services\php-8.2.29+0\bin\win64\ext" ^
  -d "extension=mbstring" ^
  C:\tmp\wp-cli.phar i18n make-pot . languages/ftb-donation-form.pot ^
  --domain=ftb-donation-form --exclude=Static,index.html --allow-root

After regenerating the .pot, reopen the .po file in Poedit and go to
Catalogue → Update from POT file to pick up the new strings.


NOTES
=====

- The wp-cli.phar file is saved at C:\tmp\wp-cli.phar (downloaded during setup)
- Locale codes follow the format language_REGION, e.g. en_US, de_DE, fr_FR
- WordPress.org requires a .pot file in /languages before the plugin is reviewed
